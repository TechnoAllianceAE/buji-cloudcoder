Engine

  • Streaming SSE API client with retry and exponential backoff
  • Model fallback on 529 (overloaded)
  • Auto-compact context when approaching token limits
  • Micro-compact for lightweight token reduction
  • Structured output via JSON schema enforcement
  • Extended thinking with budget control
  • Prompt caching headers
  • Cost tracking with per-model USD breakdown
  • Session persistence and resume
  • File history with undo/rewind
  • Git integration (repo, branch, diff detection)

Extensibility

  • Skill system — load from .bc2/skills/ with YAML frontmatter
  • Plugin system — load from .bc2/plugins/ with plugin.json manifests
  • MCP client — stdio, HTTP, and SSE transports with resource discovery
  • Hook system — pre/post tool hooks from settings.json
  • Memory system — persistent memories across sessions
  • Settings hierarchy — managed > user > project > env > defaults

Security

  • 6 permission modes (default, bypass, plan, dontAsk, acceptEdits, auto)
  • Dangerous command detection (rm -rf, git push --force, curl|sh, etc.)
  • Sensitive file protection (.env, credentials, .ssh/*, *.pem, *.key)
  • Allow/deny rules with pattern matching
  • Bridge server authentication via X-BC2-Token header
  • 38 permission tests covering all security paths

Additional Features

  • IDE bridge server (HTTP API for VS Code / JetBrains)
  • Voice engine (SoX recording + platform TTS)
  • OAuth with PKCE and auto-refresh
  • Background task manager with goroutines
  • Keybinding system (configurable, vim/emacs modes)
  • Markdown-to-ANSI renderer and HTML export
  • Feature flags system (24 flags, env-overridable)
  • Update checker (GitHub releases)
  • SSH remote sessions
  • System diagnostics (/doctor)
